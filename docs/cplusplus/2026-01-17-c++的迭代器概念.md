## 一、前言

在C++中，为实现对同质化数据序列（如容器）的通用算法操作，需设计一个轻量级访问接口。指针虽能高效遍历连续数组（通过p++移动、*p解引用），但并非所有容器都保证元素连续存储（如std::list的元素分散在内存中）。若直接依赖指针，算法将被绑定到特定容器实现。
因此，C++提出迭代器概念：只要一个类型支持++（自增）、*（解引用）和==（相等比较）这三个基础操作，它就被视为迭代器。这源于“鸭子类型”思想——无需关注具体实现，仅需满足操作契约即可。例如：

+ `int*`（指针）满足迭代器要求
+ `std::vector<int>::iterator` 也满足
+ `std::list<int>::iterator` 同样满足（但内部实现非连续）

迭代器剥离了指针的强功能（如任意偏移），仅保留遍历序列的最小必要操作。这使算法（如 `std::sort`）能统一操作任意容器，而无需关心容器内部结构，完美实现“算法依赖抽象，而非细节”的设计原则。

引用cppreference对迭代器的感性认识:

> 迭代器是对指针的泛化，它使得 C++ 程序能够以统一的方式处理不同的数据结构（例如容器和自 C++20 起引入的范围）。迭代器库提供了迭代器的定义，以及迭代器特征（iterator traits）、适配器（adaptors）和实用函数（utility functions）。由于迭代器是指针的抽象，其语义是对 C++ 中指针大多数语义的泛化。这确保了任何接受迭代器的函数模板同样也能与普通指针良好地协作。

## 二、迭代器种类

`LegacyIterator` 的要求描述了可用于标识和遍历容器元素的类型。它是迭代器最低的要求，常常把它用作是萃取一些迭代器中一些类型别名或者是标签派发。它对定义的迭代器必须满足的要求，要求主要分为三类如下：

1. 这个迭代器自身必须满足：  
   + 可拷贝构造；
   + 可拷贝复值；
   + 具有析构函数函数，以及它是可交换的。

2. 满足 `std::iterator_traits<It>`的特化要求，即迭代器内部定义如下的类型别名：  
    + `difference_type` 一般来说，它表示两个迭代器之间距离（偏移量） 的有符号整数类型。
    + `value_type` 它表示迭代器指向的元素的实际类型。
    + `pointer` 表示迭代器指向的元素的指针。
    + `reference` 表示迭代器指向的元素的引用。
    + `iterator_category` 表示迭代器所支持的具体的操作，它往往是通过标签派发的方式完成类型萃取，具体取之如下：
  
        | 迭代器能力 | C++98/03/11/14/17 标签类型（取值） | C++20 概念（Concept） |
        |-----------|-------------------------------|----------------------|
        | 输入迭代器 | `std::input_iterator_tag` | `std::input_iterator` |
        | 输出迭代器 | `std::output_iterator_tag` | `std::output_iterator` |
        | 前向迭代器 | `std::forward_iterator_tag` | `std::forward_iterator` |
        | 双向迭代器 | `std::bidirectional_iterator_tag` | `std::bidirectional_iterator` |
        | 随机访问迭代器 | `std::random_access_iterator_tag` | `std::random_access_iterator` |
        | 连续迭代器（C++20 新增） | —（无独立标签） | `std::contiguous_iterator` |

3. 如果有一个左值类型的迭代器it，那么它必须支持如下操作  
    + `value operator * ()`，他可以返回任意类型，但是一般来说，都是返回其内部定义的 `reference`。
    + `It& operator++()`，表示它可以完成自增操作。

接下来依次介绍各种类型的迭代器。

### 2.1 输出迭代器

输出迭代器往往表示一个最低的访问权限媒介，它能够自增和引用组合，从而完成对同质元素的访问，并且它是能够完成迭代器的特化的。

| 表达式      | 类型                                                                 | 语义说明 |
|-------------|----------------------------------------------------------------------|----------|
| `i != j`    | 满足 `BooleanTestable` 的类型（C++20 之前）<br>建模 `boolean-testable` 的类型（C++20 起） | **前置条件**：`i` 和 `j` 在 `==` 的定义域内。<br>**效果**：等价于 `!(i == j)`。 |
| `*i`        | 类型 `R`，可转换为 `T`                                               | **前置条件**：`i` 可解引用（dereferenceable）。<br>**效果**：<br>- 表达式 `(void)*i` 与 `*i` 等价。<br>- 若 `i` 和 `j` 在 `==` 的定义域内，且 `i == j`，则 `*i` 与 `*j` 等价。 |
| `i->m`      | —                                                                    | **前置条件**：`i` 可解引用。<br>**效果**：等价于 `(*i).m`。 |
| `++r`       | `X&`                                                                 | **前置条件**：`r` 可解引用。<br>**后置条件**：<br>- `r` 要么仍可解引用，要么处于尾后（past-the-end）位置。<br>- 所有 `r` 的先前值的副本不再要求可解引用，也不再要求处于 `==` 的定义域内。 |
| `(void)r++` | —                                                                    | **效果**：等价于 `(void)++r`。 |
| `*r++`      | 可转换为 `T` 的类型                                                  | **效果**：等价于 `T x = *r; ++r; return x;`。 |

### 2.2 输入迭代器

输入迭代器具有对这个对象具有写入的权限，它主要是对最简单的循环写入的抽象。

| 表达式         | 返回类型                | 等价表达式                     | 前置条件             | 后置条件                                                                 | 说明 |
|----------------|------------------------|-------------------------------|----------------------|--------------------------------------------------------------------------|------|
| `*r = o`       | （未使用）             | —                             | `r` 可解引用         | `r` 是可递增的（incrementable）                                           | 执行此操作后，不要求 `r` 仍可解引用；且 `r` 的任何先前值的副本不再要求可解引用或可递增。 |
| `++r`          | `X&`                   | —                             | `r` 可递增           | `r` 与 `++r` 指向同一个迭代器对象；`r` 要么可解引用，要么处于尾后（past-the-end）位置 | 执行此操作后，不要求 `r` 仍可递增；且 `r` 的任何先前值的副本不再要求可解引用或可递增。 |
| `r++`          | 可转换为 `const X&`    | ```X temp = r; ++r; return temp;``` | —                    | —                                                                        | — |
| `*r++ = o`     | （未使用）             | ```*r = o; ++r;```            | `r` 可解引用且可递增 | —                                                                        | — |

### 2.3 前向迭代器

前向迭代器是在输出迭代器的基础上，它满足所谓的“多趟保证”，即所谓的迭代器之间的自增操作是相互独立的。类型为 X 的两个可解引用迭代器 a 和 b 满足多遍（multi-pass）保证，当且仅当下列所有条件均成立：

+ 迭代器满足可默认构造。这个是在向前迭代器中新增的。
+ 若 a == b，则 ++a == ++b.
+ X 是指针类型或者是表达式 $(void)++X(a)$, $*a$ 与表达式 $*a$ 等价。

在c++中，只满足输出迭代器但是不满足前向迭代器的例子就是 `filesystem` 中的文件迭代器，它就不保证迭代器之间相互独立。

### 2.4 双向迭代器

前面的迭代器类型只满足以自增的方式实现对同质化数据进行依次访问，而双向迭代器支持通过以自减方式访问数据。即自增访问下一个，自减访问上一个。那么这样自然暗含了能够满足前向迭代器的要求，否则自减就没有太多意义了。例如，一个算法需要查看这个数据的前一个和后一个，也就是同时使用自增和自减，但是自增和自减不能影响当前这个迭代器的实际指向的对象。

| 表达式        | 返回类型                | 等价表达式                              | 说明（Notes） |
|---------------|------------------------|----------------------------------------|--------------|
| `--a`         | `It&`                  |                                        | 前提条件（Preconditions）：<br> `a` 是可递减的（即存在某个 `b`，使得 `a == ++b`）。<br>后置条件（Postconditions）：<br> `a` 是可解引用的；<br> `--(++a) == a`；<br> 若 `--a == --b`，则 `a == b`；<br> `a` 与 `--a` 指向同一个迭代器对象。 |
| `a--`         | 可转换为 `const It&`   | ```It temp = a; --a; return temp;```    | 后置递减操作符：先返回原值的副本，再将 `a` 递减。 |
| `*a--`        | `reference`            |                                        | 先对 `a` 解引用，然后执行后置递减（即解引用的是递减前的值）。 |

### 2.5 随机访问迭代器

随机访问迭代器是在双向迭代器的基础上，增加了指定不同间隔长度（间隔可正可负）去访问下一个数据的能力，而非只能依次访问数据。具体随机访问迭代器要求如下

+ `value_type`：由 `std::iterator_traits<It>::value_type` 所表示的类型。它是迭代器所指向元素的值类型（通常为去引用后对象的非引用、非 const 版本）。例如，对于 `int*`，value_type 是 int。

+ `difference_type`：由 `std::iterator_traits<It>::difference_type` 所表示的类型。
它是用于表示两个迭代器之间距离的有符号整数类型（通常为 std::ptrdiff_t）。
+ `reference`：由 `std::iterator_traits<It>::reference` 所表示的类型。
它是通过解引用迭代器（如 `*it`）所得到的引用类型。例如，对于普通指针 `int*`，`reference` 是 `int&`；对于 `const int*`，则是 `const int&`。
+ `i, a, b`：类型为 `It` 或 `const It` 的对象；`r`：类型为 `It` 的左值（lvalue）；`n`：类型为 `difference_type` 的整数

| 表达式 | 返回类型 | 操作语义（Operational Semantics） | 说明（Notes） |
|--------|----------|----------------------------------|----------------|
| `r += n` | `It&` | ```cpp<br>difference_type m = n;<br>if (m >= 0) while (m--) ++r;<br>else while (m++) --r;<br>return r;<br>``` | - `n` 可正可负。<br>- 复杂度必须是常数时间（即实际实现不能真的执行上述 `while` 循环，仅用于语义说明）。 |
| `a + n`<br>`n + a` | `It` | ```cpp<br>It temp = a;<br>return temp += n;<br>``` | - `n` 可正可负。<br>- 满足交换律：`a + n == n + a`。 |
| `r -= n` | `It&` | `return r += -n;` | - 要求 `|n|` 在 `difference_type` 可表示范围内。 |
| `i - n` | `It` | ```cpp<br>It temp = i;<br>return temp -= n;<br>``` | — |
| `b - a` | `difference_type` | `return n;`（见前提条件） | 前提条件：<br>存在一个 `difference_type` 类型的值 `n`，使得 `a + n == b`。<br>后置条件：<br>`b == a + (b - a)`。 |
| `i[n]` | 可转换为 `reference` | `*(i + n)` | 等价于偏移后解引用。 |
| `a < b` | 满足 `BooleanTestable`（C++20 前）<br>建模 `boolean-testable`（C++20 起） | 等价于 `b - a > 0` | 前提条件：同 `b - a`。<br>严格全序关系（Strict total ordering）：<br>• `!(a < a)`（反自反）<br>• 若 `a < b`，则 `!(b < a)`（反对称）<br>• 若 `a < b` 且 `b < c`，则 `a < c`（传递）<br>• `a < b`、`b < a`、`a == b` 中有且仅有一个为真。 |
| `a > b` | 同上 | `b < a` | 与 `<` 相反的全序关系。 |
| `a >= b` | 同上 | `!(a < b)` | — |
| `a <= b` | 同上 | `!(a > b)` | — |

### 2.6 连续访问迭代器

随机访问迭代器只是说支持以不同间隔去访问数据，但是它并没有要求这个数据在实际物理内存上是如何存储的。连续访问迭代器并没有在随机访问迭代器的基础上进行定义，而是直接在迭代器要求上进行定义的。

类型 It 满足 LegacyContiguousIterator（传统连续迭代器）的条件是：

+ 类型 It 满足 LegacyIterator（传统迭代器）的要求；
+ 对于任意 a：一个类型为 It 的可解引用迭代器， n：一个整数值，  
    只要 a + n 是一个有效且可解引用的迭代器值，那么表达式 `*(a + n)` 和 `*(std::addressof(*a) + n)` 是等价的。

## 三、迭代器实现过程

第一步，满足 LegacyIterator（最低要求）

```c++

template<typename T, bool IsConst>
struct IteratorImpl {
    using value_type = typename std::conditional<IsConst, const T, T>::type;
    using reference = value_type &;
    using pointer = value_type *;
    using difference_type = std::ptrdiff_t;
    using iterator_category = std::random_access_iterator_tag;
    reference operator*(); // 返回值不一样是reference

    IteratorImpl(const IteratorImpl & ) = default;
    operator = (const IteratorImpl & ) = default;
    ~IteratorImpl() = default;
    void swap(IteratorImpl & );
};
```

第二步，升级为输入迭代器，要求解引用返回值必须是 `reference` ，并且新增访问运算符。

```c++
pointer operator->() const { return &(*(*this)); }
bool operator!=(const IteratorImpl &other) const;
IteratorImpl & operator++();
IteratorImpl operator++(int);
```

第三步，升级为向前迭代器，这个没办法通过一些函数约束或者是类型别名要求，只能是具体实现是否满足，往往可以通过单元测试实现，拥有默认构造函数。

```c++
IteratorImpl()=default；
```

第四步，升级为双向迭代器，在 `LegacyForwardIterator` 基础上，支持自减操作 `（--it 和 it--）`。

```c++
IteratorImpl &operator--() { ... }  // 前缀自减
IteratorImpl operator--(int) { ... }                    // 后缀自减
```

第五步，升级为随机访问迭代器，主要是增加对随机偏移的支持。

```c++
IteratorImpl operator+(difference_type n) const { ... }  // 随机偏移
IteratorImpl operator-(difference_type n) const { ... }  // 反向偏移
difference_type operator-(const IteratorImpl &other) const { ... }  // 距离计算
reference operator[](difference_type n) const { ... }  // 随机访问
bool operator<(const IteratorImpl &other) const { ... }  // 比较
friend IteratorImpl operator+(difference_type n, const IteratorImpl &it);
IteratorImpl operator+(difference_type n) const;
IteratorImpl operator-(difference_type n) const;
IteratorImpl &operator+=(difference_type n);
IteratorImpl &operator-=(difference_type n);
bool operator==(const IteratorImpl &other) const;
bool operator<(const IteratorImpl &other) const;
bool operator<=(const IteratorImpl &other) const;
bool operator>(const IteratorImpl &other) const;
bool operator>=(const IteratorImpl &other) const;
```
