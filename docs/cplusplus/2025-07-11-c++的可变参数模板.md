## 一、前言

可变参数模板是c++11提出的一种增强泛型能力的技术。首先看看一般模板具有哪些限制，或者是哪些功能是开发者想要的，但是语言标准并不支持。假设我们想要实现一个求和算法，它支持满足各自相加算法的类相互加在一起，并且接收任意数量的参数，这些参数类型并不一定一样。那么就像如下形式：

```c++
template<typename T> 
T sum(T val) {
    return val;
}

template<typename T, typename T2> 
T sum(T val, T2 val2) {
    return val + val2;
}
// ...
template<typename T, typename T2, ... typename Tn> 
T sum(T val, T2 val2, ... Tn valn) {
    return val + val2 + ... valn;
}
```

一种朴素的实现就是手动打出任意数量的模板参数，然后手动实现求和。对于想偷懒的程序员来说，这并不是一个很好的实现方式。

那么假设c++标准想要给模板提供这种不定参数的能力，那么怎么把需求转化为现有标准改动较小，又能完整提供实现能力。先看看怎么把这个可变参数这个功能具体描述成问题：

1. 这个模板能够实现接收任意个数的模板参数。
2. 程序员可以操作这一堆模板参数或者是实际传入的参数，或者换句话说，具有依次操作这堆参数中任意一个参数的接口。
3. 程序员有能力知道这对参数的个数是多少。

那么c++标准提出的使用“包”（pack）的概念。

## 二、pack--包

根据cppreference的定义：
>A pack is a C++ entity that defines one of the following:
> + a parameter pack
>   - template parameter pack
>   - function parameter pack
> + lambda init-capture pack (since C++20)
> + structured binding pack (since C++26)

也就是说：一个包是c++中的一个实体，在以下情况中都被认为是一个包：

+ 一个参数包
    - 模板参数包
    - 函数参数包
+ lambda初始化捕捉的包
+ 结构化绑定包

后面两种是c++较为新的概念，此处先看第一个概念，参数包。

模板参数包原文对其的定义为：`A template parameter pack is a template parameter that accepts zero or more template arguments (constants, types, or templates).`，首先说模板参数包是一个模板参数，然后这个模板参数可以接受0至任意个模板参数，这些模板参数可以是常量，类型，或者是模板。再看函数参数包的定义：`A function parameter pack is a function parameter that accepts zero or more function arguments.`，函数参数包也是一个函数参数，然后它也能够接收0至任意个函数参数。

可以回到前面需求端的第一需求，它的定义和我们的需求是一致的，也就是说c++规定接收任意个参数的东西定义为参数包的东西，然后定义两种具体的类型分别叫模板参数包和函数参数包。那么对于第二个需求，cppreference说，如果一个包是模板参数包或者是函数参数包的话，它负责提供参数包中具体有多少个参数。

接着定义了可变参模板的定义：如果一个模板有至少一个参数包，那么它就是可变参模板。

前面的内容只是一堆定义，表示了可变参模板是什么，以及将前面的程序员的需求以标准的方式定义了，其实就是规定名词。具体还是写成代码，我们介绍其具体表现形式。

### 2.1 参数包表现形式

首先介绍模板参数包表现形式：

```c++
// 1. 非模板参数且只有一个参数包的情况
template<type ... TemplatePackName> 

// 2. 模板参数且只有一个参数包的情况
template<typename ... TemplatePackName> 

// 3. 其他参数和可变参数包混合
template<typename T，std::size_t N, typename ... TemplatePackName> 

```

可以看出c++对参数包的在代码中的表现形式更像是可变参数宏的声明方式，即使用typename或者是具体类型，接着省略号，然后定义包名。

函数参数包是定义可变参函数模板出现的，它的具体表现形式为：

```c++
template<typename ... TemplatePackName> 
void func1(TemplatePackName... func_para_ame)
{
    return;
}
```

相比于普通模板函数，就是将上面的模板参数包加了一个省略号，然后给这个参数包给一个名字。

__总结：__

1. 模板参数包就是加了省略号，给一个名称TemplatePackName，当然TemplatePackName可以任意取，用这个标识这个模板参数包。
2. 函数参数包就是将模板定义中的包名放到函数参数列表中，然后后面加省略号，然后给一个名称标识这个函数参数包

这一节只是表示咋样算定义一个可变参模板，那怎么操作它呢，c++标准提出使用包展开"pack expansion"。

### 2.2 参数包展开

首先，参数包展开只能位于模板体内。语法为

```c++
{
    pattern(packname) ...;
}
```
具体作用效果为：根据扩展发生的位置，生成的逗号分隔列表是一种不同类型的列表：函数参数列表、成员初始化器列表、属性列表等。其中参数包展开的时候，align修饰符是以空格分开的。像是如下效果

```
//编译器读取出来的参数包实际内容：args = {int, char, double, std::string};

// 用户填写的，parttern(args)...

// 编译器生成的，parttern(int),parttern(char),parttern(double),parttern(std::string)
```

### 2.3 参数包展开位置

具体作用效果根据位置稍微有一些不一样：

1. 函数实参参数列表中：包扩展出现在函数调用运算符的括号内，在这种情况下，省略号左侧最大的表达式或括号括起来的初始化器列表是扩展的模式
    ```c++
    f(args...);              // expands to f(E1, E2, E3)
    f(&args...);             // expands to f(&E1, &E2, &E3)
    f(n, ++args...);         // expands to f(n, ++E1, ++E2, ++E3);
    f(++args..., n);         // expands to f(++E1, ++E2, ++E3, n);
    ```
2. 带圆括号的初始化器：包扩展可能出现在直接初始化器、函数样式转换和其他上下文（成员初始化器、新表达式等）的括号内，在这种情况下，规则与上述函数调用表达式的规则相同
    ```c++
    Class c1(&args...);             // calls Class::Class(&E1, &E2, &E3)
    Class c2 = Class(n, ++args...); // calls Class::Class(n, ++E1, ++E2, ++E3);
    
    ::new((void *)p) U(std::forward<Args>(args)...) // std::allocator::allocate
    ```
3. 花括号的统一初始化器：这种情况要求参数都是一样，那么实现了不同类型参数的构造函数。这个可参考初始化方式文章。
    
    ```c++
    template<typename... Ts>
    void func(Ts... args)
    {
        const int size = sizeof...(args) + 2;
        int res[size] = {1, args..., 2};
    
        // since initializer lists guarantee sequencing, this can be used to
        // call a function on each element of a pack, in order:
        int dummy[sizeof...(Ts)] = {(std::cout << args, 0)...};
    }
    ```
4. 模板形参参数列表中：
    ```c++
    template<class A, class B, class... C>
    void func(A arg1, B arg2, C... arg3)
    {
        container<A, B, C...> t1; // expands to container<A, B, E1, E2, E3> 
        container<C..., A, B> t2; // expands to container<E1, E2, E3, A, B> 
        container<A, C..., B> t3; // expands to container<A, E1, E2, E3, B> 
    }
    ```
5. 函数形参参数列表中：
    ```c++
    template<typename... Ts>
    void f(Ts...) {}
    
    f('a', 1); // Ts... expands to void f(char, int)
    f(0.1);    // Ts... expands to void f(double)
    
    template<typename... Ts, int... N>
    void g(Ts (&...arr)[N]) {}
    
    int n[1];
    
    g<const char, int>("a", n); // Ts (&...arr)[N] expands to 
                                // const char (&)[2], int(&)[1]
    ```

6. 模板形参列表中：
    ```c++
    template<typename... T>
    struct value_holder
    {
        template<T... Values> // expands to a constant template parameter 
        struct apply {};      // list, such as <int, char, int(&)[5]>
    };
    ```
7. 基类标识符和成员初始化列表中：
    + 包拓展可以指定类声明中的基类列表，也就是说可以将一些类传递给它，然后它自动继承这些
        ```c++
        template<class... Mixins>
        class X : public Mixins...
        {
        public:
            X(const Mixins&... mixins) : Mixins(mixins)... {}
        };

        x<int, std::string> ff; // x : public int, std::string
        // 虽然不知道有啥明显的用途，但是c++提供了这种能力
        ```
8. Lambda 捕捉：包扩展可能出现在lambda表达式的捕获子句中：
    ```c++
    template<class... Args>
    void f(Args... args)
    {
        auto lm = [&, args...] { return g(args...); };
        lm();
    }
    ```
9. Alignment表示符，它是用空格隔开的，这个比较特殊
    ```c++
    template<class... T>
    struct Align
    {
        alignas(T...) unsigned char buffer[128];
    };
    
    Align<int, short> a; // the alignment specifiers after expansion are
                        // alignas(int) alignas(short)
                        // (no comma in between)
    ```
10. 属性列表展开：如果属性规范允许，则允许在属性列表中进行包扩展。
    ```
    template<int... args>
    [[vendor::attr(args)...]] void* f();
    ```

### 2.4 参数包展开细则

前面说明了参数包展开的样式，现在讲讲其他规则：

1.如果两个包的名称以相同的模式出现，则它们将同时展开，并且它们必须具有相同的长度：
    ```c++
    template<typename...>
    struct Tuple {};
    
    template<typename T1, typename T2>
    struct Pair {};
    
    template<class... Args1>
    struct zip
    {
        template<class... Args2>
        struct with
        {
            typedef Tuple<Pair<Args1, Args2>...> type;
            // Pair<Args1, Args2>... is the pack expansion
            // Pair<Args1, Args2> is the pattern
        };
    };
    
    typedef zip<short, int>::with<unsigned short, unsigned>::type T1;
    // Pair<Args1, Args2>... expands to
    // Pair<short, unsigned short>, Pair<int, unsigned int> 
    // T1 is Tuple<Pair<short, unsigned short>, Pair<int, unsigned>>
    
    // typedef zip<short>::with<unsigned short, unsigned>::type T2;
    // error: pack expansion contains packs of different lengths
    ```
2. 如果包扩展嵌套在另一个包扩展中，则出现在最内层包扩展中的包将被它扩展，并且必须在封闭的包扩展中提到另一个包包，但不在最内层的包中：
    ```c++
    template<class... Args>
    void g(Args... args)
    {
        f(const_cast<const Args*>(&args)...); 
        // const_cast<const Args*>(&args) is the pattern, it expands two packs
        // (Args and args) simultaneously
    
        f(h(args...) + args...); // Nested pack expansion:
        // inner pack expansion is "args...", it is expanded first
        // outer pack expansion is h(E1, E2, E3) + args..., it is expanded
        // second (as h(E1, E2, E3) + E1, h(E1, E2, E3) + E2, h(E1, E2, E3) + E3)
    }
    ```
3. 当包中的元素数量为零时（空包），包扩展的实例化不会改变封闭构造的语法解释，即使在完全省略包扩展会导致格式错误或导致语法歧义的情况下也是如此。实例化生成一个空列表。
    ```c++
    template<class... Bases> 
    struct X : Bases... { };
    
    template<class... Args> 
    void f(Args... args) 
    {
        X<Args...> x(args...);
    }
    
    template void f<>(); // OK, X<> has no base classes
                        // x is a variable of type X<> that is value-initialized
    ```

### 2.5 总结

参数包展开加pattern(arg)...实际上提供给程序员单个处理参数包中每个元素的能力，如果想要负责处理的话，比较麻烦一些。

在c++26中有了包索引“Pack indexing”的提案。

## 三、获取参数包数量

因为这个参数包是一个编译期实体，那么c++提供获取数量的方式是sizeof运算符。

```c++
template<class... Types>
struct count
{
    static const std::size_t value = sizeof...(Types);
};
```

相较于普通的sizeof变量，sizeof参数包是多了一个省略号。

## 四、 三个需求解决方案

总体上c++标准通过像是高级文本替换的方式，在编译期动态地生成了一堆函数或者是类定义以及相关定义，实现了可变参的能力。后面在c++14中，提出了编译器整数序列，能够在一定程度上提高对单个参数的处理能力。
