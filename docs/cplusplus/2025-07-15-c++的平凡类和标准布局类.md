## 一、前言

c++中为了支持类的各种特性，很多时候会对这个类进行一定的修饰，这些修饰很多时候并没有在标准中约束，但是c++是一门支持操作底层数据的语言，那么在操作内存时，往往预先需要确定这个类在内存中到底是怎么表示的，否则程序就没办法按照预期的行为运行。为了将类的实例——即对象在内存中进行一定的约束，c++11提出来三种概念：

1. 可平凡拷贝构造类（Trivially copyable class）
2. 平凡类（Trivial class）
3. 标准布局类（Standard-layout class）

这三个概念各自对类提出了一定的要求，当一个类分别满足这三个条件时，也就意味着一些操作是不会出现未定义行为。首先我们先根据其各自的定义来确定它想要支持什么操作，从而就明白了它为什么要这么定义。

## 二、可平凡拷贝构造类

### 2.1 定义

一个类满足如下所有条件后，便可以称为一个平凡拷贝类：

1. 拥有至少一个可使用的拷贝函数、移动构造函数、拷贝赋值运算符、或者是一个移动运算符。
> has at least one eligible copy constructor, move constructor, copy assignment operator, or move assignment operator,
    + 这句话看起来像是只要这四个特殊函数存在一个即可，可实际上，测试下来是只要带cv限定符版本移动运算符和cv限定符号版本的移动构造函数删除了，它就认为不是可拷贝构造的了。也就是说至少它的移动构造不能受到影响。至于为什么，我也不明白。
        ```c++
        struct BaseA {
            int a; 
            // 1. 彻底禁止默认构造
            BaseA() = delete;

            // 2. 禁止所有拷贝构造变体
            BaseA(BaseA&) = delete;                     // non-const 版本
            BaseA(const BaseA&) = delete;               // const 版本 (标准)
            BaseA(volatile BaseA&) = delete;            // volatile 版本
            BaseA(const volatile BaseA&) = delete;      // const volatile 版本

            // 3. 禁止所有移动构造变体
            BaseA(BaseA&&) = delete;                    // non-const 版本
            // BaseA(const BaseA&&) = delete;              // const 版本
            // BaseA(volatile BaseA&&) = delete;           // volatile 版本
            // BaseA(const volatile BaseA&&) = delete;     // const volatile 版本

            // 4. 禁止所有赋值运算符变体
            BaseA& operator=(BaseA&) = delete;
            BaseA& operator=(const BaseA&) = delete;
            BaseA& operator=(volatile BaseA&) = delete;
            BaseA& operator=(const volatile BaseA&) = delete;
            BaseA& operator=(BaseA&&) = delete;
            // BaseA& operator=(const BaseA&&) = delete;
            // BaseA& operator=(volatile BaseA&&) = delete;
            // BaseA& operator=(const volatile BaseA&&) = delete;
        };
        ```
2. 每一个合格的拷贝构造函数是平凡的，对于拷贝构造函数而言，平凡的定义如下：
    + 它不是用户提供的，换句话说，它要么不写，隐式生成，要么是使用default关键字生成的。
    + 它没有虚函数
    + 他没有虚基类
    + 它的每一个直接继承的父类的所选择的拷贝构造函数都是平凡的
    + 它的非静态数据成员的拷贝构造函数是平凡的。

3. 每一个合格的移动构造函数是平凡的，对于移动构造函数而言，平凡的定义如下：
    + 它不是用户提供的，换句话说，它要么不写，隐式生成，要么是使用default关键字生成的。
    + 它没有虚函数
    + 他没有虚基类
    + 它的每一个直接继承的父类的所选择的移动构造函数都是平凡的
    + 它的非静态数据成员的移动构造函数是平凡的。
4. 每一个合格的拷贝赋值运算符是平凡的，对于拷贝赋值运算符而言，平凡的定义如下：
    + 它不是用户提供的，换句话说，它要么不写，隐式生成，要么是使用default关键字生成的。
    + 它没有虚函数
    + 他没有虚基类
    + 它的每一个直接继承的父类的所选择的拷贝赋值运算符都是平凡的
    + 它的非静态数据成员的拷贝赋值运算符是平凡的。
5. 每一个合格的移动赋值运算符是平凡的，对于拷贝赋值运算符而言，平凡的定义如下：
    + 它不是用户提供的，换句话说，它要么不写，隐式生成，要么是使用default关键字生成的。
    + 它没有虚函数
    + 他没有虚基类
    + 它的每一个直接继承的父类的所选择的移动赋值运算符都是平凡的
    + 它的非静态数据成员的移动赋值运算符是平凡的。
6. 有一个未被删除的平凡析构函数，对于析构函数而言，平凡的定义如下：
    + 它不是用户提供的，换句话说，它要么不写，隐式生成，要么是使用default关键字生成的。
        - 这里只说了不是用户定义，那么如果是使用delete删除了呢或者是private呢，结果是断言成功。
            ```c++
            struct A{
                int dd;
                ~A() = delete;
            };
            static_assert(std::is_trivially_copyable_v<A>, "Must stay trivial");

            struct B{
                int dd;
            private:
                ~B() = default;
            };
            static_assert(std::is_trivially_copyable_v<B>, "Must stay trivial");
            ```
        - 写了声明之后后面没有default或者是delete关键字修饰的话，会认为是用户自定义，这个断言失败。
    + 它不是虚析构的
    + 他没有虚基类
    + 它的每一个直接继承的父类的析构函数是平凡的
    + 它的非静态数据成员的析构函数是平凡的。

可以总结如下：
    
1. 有移动构造能力，至于是否具有拷贝构造能力，不一定。
2. 没有虚函数或者是虚继承。
3. 析构函数是默认生成的。
4. 基类也是一个平凡类。
5. 非静态数据成员也是平凡类。

### 2.2 意义

平凡复制类是要求提供一种运行时的自包含状态。就是说，当程序运行起来之后，如果创建了一个平凡类的实例化对象a，那么只需要内存拷贝memcpy一个新的地址中，比如对象b，那么这两个对象所表示的状态是完全一致的。个人感觉为啥要取名平凡类，就是想要表达像c语言那样的，只需要内存拷贝就可以把一个对象复制给另外一个对象。那么自包含又是啥意思呢？一般情况下，c++对象中不仅仅包含了其内部的数据成员，也包含引用自其他地方的资源，这个资源可能是堆上的资源，也可能是其他地方的资源，那么平凡复制类因为其默认析构函数，那么它的所有关于数据描述的部分，都没有使用raii进行资源管理，也就是说，这个资源的所有权与其内部状态无关。那么根据上面的要求，可感性地总结如下：

1. 有移动或者是复制构造能力，表示这个类是一种可复制地状态，也就是说，通过内存拷贝可以达到重复、批量化地制造对象地感觉。
2. 没有虚函数或者是虚继承，根据微软教程，这个应该是必须这些增强类表达能力地东西将一个对象的内存资源不连续，也就是说有且仅有一块连续的内存就可以完整的表达一个对象。
3. 析构函数是默认生成的，这个表达的是对象内部不拥有外部资源所有权，而仅仅是读取的权力。
4. 要求所有组成数据成员的部分，比如基类和非静态数据成员都满足上面的1~3的条件。

微软官方对于普通类型的感性要求：[平凡类](https://learn.microsoft.com/zh-cn/cpp/cpp/trivial-standard-layout-and-pod-types?view=msvc-170)

> 当 C++ 中的类或结构具有编译器提供的或显式默认设置的特殊成员函数时，该类或结构为普通类型。 它占用连续内存区域。 它可以具有含不同访问说明符的成员。 在 C++ 中，编译器可以自由选择在此情况下对成员排序> 的方式。 因此，你可以在内存中复制此类对象，但不能从 C 程序中可靠地使用它们。 可以将普通类型 T 复制到 char 或无符号 char 数组，并安全地复制回 T 变量。 请注意，由于对齐要求，类型成员之间可能存在填> 充字节。



## 三、平凡类

### 3.1 定义

平凡类是拥有平凡拷贝的能力，并且至少有一个默认构造函数，而这个默认构造函数是平凡的。对于默认构造函数而言，它的平凡性定义如下：

1. 默认构造函数时隐式生成或者是使用default生成的。
2. 没有虚函数或者有虚继承。
3. 每一个非静态数据成员都没有默认初始化器，也就是说数据成员没有默认值。
    ```c++
    struct a {
        int b {0}; // 不行
        int c = 0; // 不行
    };
    ```
4. 每一个当前继承的父类都有一个默认构造函数（Every direct base of T has a trivial default constructor.）
5. 每一个非静态数据数据成员都有一个默认构造函数，实际上它也递归的要求它的类满足前面的1~3条，虚函数和虚继承中在可平凡拷贝中已经要求了，这里只说明默认初始化器。
    + 对于初始化器而言
        ```c++
        struct base{
            int tt = 0;
        };

        struct a {
            base fff;
            int c;
        };
        static_assert(std::is_trivial_v<base>, "Must stay trivial");//断言失败
        ```
6. 一旦T是一个union类型，或者是每一个非variant的非静态数据成员都有一个默认构造函数。

感性上来说，平凡类实际上是在可平凡拷贝类上要求实现默认构造函数是编译器自动生成的，并且所有数据成员部分，无论是从继承得来的，还是类内声明数据成员，都不能有初始值。

### 3.2 意义

可能进一步限制了类的声明吧，至于为啥要这样限制，个人感觉是避免通过引用或者是指针的获取外部资源的能力，因为引用必须初始化的时候就使用，如果是平凡默认构造的话，应该表示这个类不具有初始化过去外部数据的能力。

## 四、 标准布局类

根据微软官方，标准布局类主要是表示没有c++功能的一些类。实际感觉是，它想要实现c++的二进制abi像c语言结构体那样的结构稳定，像是堆方块一样，把所有的数据都堆叠起来。
> 当类或结构不包含某些 C++ 语言功能（例如无法在 C 语言中找到的虚拟函数），并且所有成员都具有相同的访问控制时，该类或结构为标准布局类型。 可以在内存中对其进行复制，并且布局已经过充分定义，可以由 C 程序使用。 标准布局类型可以具有用户定义的特殊成员函数

### 4.1 定义

标准布局类满足如下条件：

1. 没有引用，并且静态数据成员的类或者是数组的类型是标准布局的。
2. 没有虚函数和虚继承。
3. 非静态数据成员具有相同的访问权限。同为public，protected以及private
4. 没有标准布局的基类
5. 在继承链中只有一个类拥有非静态数据成员（only one class in the hierarchy has non-static data members），换句话说，最底层派生类中没有非静态数据成员，并且具有非静态数据成员的基类不超过一个，或者没有含非静态数据成员的基类。那么这里的层次结构是指继承链都是树状的还是说可以有菱形继承呢？毕竟按照原文中只强调了是否有数据，并没有强调继承链是如何的。
    + 被菱形继承的基类有数据成员
        ```c++
        struct base {
            int c;
        };

        struct base1:public base {
        };

        struct base2:public base {
        };

        struct base3:public base1, base2 {
        };
        static_assert(std::is_standard_layout_v<base3>, "Must stay standard_layout"); //断言失败
        ```
    + 没有数据成员的菱形继承
        ```c++
        struct base {
        };

        struct base1:public base {
        };

        struct base2:public base {
        };

        struct base3:public base1, base2 {
        };
        static_assert(std::is_standard_layout_v<base3>, "Must stay standard_layout"); //断言失败
        ```
    + 总结：想要标准布局的话，继承的类不能够是菱形继承。
6. 类中第一个非静态类型与基类不是同一个类型（none of the base classes has the same type as the first non-static data member.）这给人感觉是只要继承的类不是第一个数据成员即可。精确的来说，假设给定一个类S，那么一个集合作用函数M，它的作用是提取类S的第一个数据成员的类型，以及递归提取第一个数据成员的第一个数据成员的类型，直到数据成员是基础类型为止。只要继承链中没有出现在第一个M(S)中即可。另外一个问题是如果第一个数据成员x中继承了类S继承链中存在的有数据的地方，会不会有问题呢？分两种情况
    + 如果被继承的类的继承链中没有数据成员，那么是被认为是标准布局
        ```c++
        struct base {
            // int c;
        };

        struct base1:public base {
        };

        struct data : public base{
        };
        struct base3:public base1 {
            data tt;
        };

        /*
        数据继承了base，然后base3的有一个数据成员data，又同时继承了base1.
        */
        ```
    + 如果被继承的类的继承链中有数据成员，那就不是标准布局，如下所示。
        ```c++
        struct base {
            int c;
        };

        struct base1:public base {
        };

        struct data : public base{
        };
        struct base3:public base1 {
            data tt;
        };

        /*
        数据继承了base，然后base3的有一个数据成员data，又同时继承了base1.
        */
        ```

### 4.2 意义

标准布局类并没有限制继承的能力，而标准布局类想要表明的是一种类结构的不变性，即使用不同版本的编译器如果都是标准布局类的话，描述这个类的内存应该都是一样的，这也就是说存在着二进制兼容性。假如一个类，它没有c++的特性，如继承等，那么也就保证了其二进制兼容，从而像c语言那样使用了同样的堆叠方式。只不过c++标准并没有将继承这个能力从标准布局剔除，它应该是想要表达一种情况，简单的树状继承，并且整个继承链上只有一个地方有数据这种朴素的构造方式。

实际上，根据上面的定义，cppreference对于如何精准描述标准布局类，个人认为有一定的描述不清的地方。所以工程上想要使用标准布局性质，然后利用c结构体跟其他语言交互的话，最好不要使用继承，就使用组合的方式来组织数据。一旦引入了继承，就难界定这到底算不算标准布局类。

实际上，一般应用中，标准布局类往往用于其他数据的交互，这种情况下，常常结合平凡类的性质，构造出较为良好的类性质。

## 五、聚合类

聚合类，一个朴素的感觉就是可以在c语言中定义的结构体或者是联合体。

### 5.1 定义

聚合类当且仅当如下条件满足时：

1. 它是一个平凡类。
2. 它是一个标准布局类。
3. 它的非静态成员数据是聚合类。 

### 5.2 应用

聚合类一般都是表达和c语言兼容的类。所以具体使用的时候，通过c语言编译器编译看看是否能够正确编译即可。


## 六、后记

从平凡类、到标准布局类、再到聚合类，它们主要是更方便的工程应用。平凡类更多描述的是运行期使用memcpy即可转移类的所有状态，而标准布局是讲述了如何完成类内部的兼容。聚合类更多是用来跨语言交互。这当然可以构造出一些很复杂，很刁钻的、满足条件的类，但这不是搬起石头砸自己的脚嘛。

构造一个类时，如果是数据交互，如果能够设计成满足平凡类的性质的类，那么其转移效率必然更高。

引用[C++11 内存布局术语解读（平凡类型、平凡可复制类型、标准布局类型）](https://blog.csdn.net/vviccc/article/details/138089766)，提出这些类型的动机是什么:

>下面用通俗的语言解释这些概念：

> + 平凡类型：指那些在内存中的行为非常简单的类。它们的构造函数、析构函数、拷贝构造函数和赋值运算符都没有自定义实现，完全由编译器提供的默认行为即可。这意味着这些类的对象可以像基本数据类型一样被创建和销毁，不需要特殊的资源管理代码。
> + 平凡可复制类型：是平凡类型的一个扩展，它不仅包括所有平凡类型，还包括那些可以安全地被复制和移动的类型，即使这些类型不是平凡类型。例如，一个类可能有一个自定义的构造函数，但如果它保证对象的内容可以通过简单的位拷贝（bitwise copy）来复制，那么它也可以被认为是平凡可复制的。
> + 标准布局类型：指那些在内存布局上满足一定规则的类。这些规则包括所有非静态成员的访问权限必须相同，类不能有虚函数或虚基类，且所有基类也必须是标准布局类型。标准布局类型的一个重要特性是它们的内存布局在不同的编译器和平台上是一致的，这对于跨平台的二进制数据交换非常重要。
> 接下来用更加通俗和易于理解的方式来说明，解释如下：

> + 平凡类型：就像一个简单的储物箱，没有锁，没有特殊的开启方式，任何人都可以简单地打开和关闭。
> + 平凡可复制类型：类似于平凡类型，但是更加灵活，即使储物箱有锁，只要我们保证复制时锁的状态不变，它也能满足要求。
> + 标准布局类型：就像一个有统一标准的书架，无论在哪个图书馆，书架上书的排列方式都是一致的，方便管理和查找。
