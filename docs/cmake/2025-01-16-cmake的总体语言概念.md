## 一、前言  

cmake是通过读取文件内容，以解释器的方式运行。使用者是通过文件来告诉cmake如何进行工作。

## 二、文件组织方式  

为了更好的规范接受的文件，cmake提出来三种类型的文件：

+ Directories(CMakeLists.txt) 顶层目录文件  
+ Scriptions(Scriptions.cmake) 脚本文件  
+ Modules(module.cmake) 模块文件  

可以看到，cmake仅接受三种文件，一种是固定名称CMakeLists.txt，这个命名不可更改。另外两种是自定义名称，但是需要以cmake的文件拓展名标识。  

### 2.1 Directories

当 CMake 处理一个项目的源代码树时，入口点是顶级源代码目录中的一个名为 CMakeLists.txt 的源文件。这个文件可能包含整个构建规范，或者使用 add_subdirectory() 命令将子目录添加到构建中。每个通过该命令添加的子目录也必须包含一个 CMakeLists.txt 文件作为该目录的入口点。对于每个源代码目录，CMake 会处理其 CMakeLists.txt 文件，并在构建树中生成一个相应的目录，该目录将作为默认的工作目录和输出目录。  

值得注意的点：  

1. 每个 CMakeLists.txt 都有自己的作用域。变量定义在一个目录中，对该目录以及所有子目录是可见的，但对父目录或兄弟目录不可见。  
2. 当CMake处理项目时，会根据有序的目录结构生成对应的构建树。这可以在构建目录（通常由用户指定）中看到。

### 2.2 脚本文件

一个单独的 `<script>.cmake` 源文件可以通过使用命令行工具 `cmake` 并带上 `-P` 选项以脚本模式来处理。脚本模式仅仅运行给定的 CMake 语言源文件中的命令，并不会生成构建系统。同时，脚本模式下不允许使用那些定义构建目标或行为的 CMake 命令  

### 2.3 模块文件

CMake 语言代码无论是在目录还是脚本中，都可以使用 `include()` 命令在包含上下文的作用域中加载 `<module>.cmake` 源文件。关于include命令，可参考之前的文章。在自己的项目中，我们也可以自己编写相应的module.cmake，并将这些文件统一放置到项目的一个目录下，然后通过 `CMAKE_MODULE_PATH` 变量中指定它们的位置。  

## 三、语法

### 3.1 文件编码格式  

cmake默认按照七位的ascii码，可以在大部分平台使用，并且他会根据平台的方式来确定文件读取格式。因为ascii码主要表示英文单词，因此变量名称等东西尽量使用英语字母编写。  

### 3.2 cmake的源文件  

cmake的源文件是由若干个命令调用，命令与命令之间是通过换行符区分。  

### 3.3 cmake命令参数解析规定  

cmake的命令调用是以空格作为区分不同的参数。  

### 3.4 命令参数  

命令调用的参数有三种：

1. 括号参数。使用一对双中括号包围的参数， `[[...]]` ，用于多行字符串或者包含特殊字符的情况。中括号之间的内容将原样保留，不解析其中的字符串。用于多行字符串或需要保留原格式的参数  
2. 引用参数。使用双引号包围的参数。`"..."`，双引号内的内容作为一个整体处理，但可以使用转义字符来包含特殊字符或者嵌入新的引号。用于包含空格或特殊字符的字符串  
3. 非引用参数。没有使用括号或引号包围的参数。这种参数可以包含标点符号、顶层的未加引号的括号等。非引号参数在遇到空格或行结束符时结束。用于简单的、不包含空格的字符串或标识符。  

### 3.5 转义序列 

使用'\'对特殊字符进行转义处理。

### 3.6 使用变量

`${VAR_NAME}`: 这是最基本的变量引用语法，表示引用一个名为 `VAR_NAME` 的变量的值。

使用set(VAR_NAME "xx")定义初始变量或者是重新赋值变量。  

## 四、控制结构

1. if()/elseif()/else()/endif() 条件结构。  
2. foreach()/endforeach() 和 while()/endwhile()。使用break()和continue()控制。
3. 命令定义。The macro()/endmacro(),和 function()/endfunction()。用于封装代码块并提高复用性。函数有局部作用域，而宏没有。宏没有自己的作用域。即在宏内部定义的变量是全局的。宏内部定义的变量会影响到外部变量。

## 五、变量

变量是CMake语言中的基本存储单位。它们的值总是字符串类型的，尽管有些命令可能会将字符串解释为其他类型的值。set（）和unset（）命令显式设置或取消设置变量，但其他命令也有修改变量的语义。变量名区分大小写，几乎可以由任何文本组成，但建议坚持使用仅由字母数字字符加_和-组成的名称。

### 5.1 作用域

CMake中的变量作用域和持久缓存机制是CMake脚本语言的重要特性。

1. Block Scope。
      + Block Scope是通过 `block()` 命令创建的新作用域。
      + 使用 `block()` 允许在代码块中创建新的变量绑定作用域。
      + 具体来说，所有在 `block()` 和 `endblock()` 之间定义的变量在块结束后会释放，不会影响块外的变量。

2. Function Scope。
      + 通过 function() 命令定义的命令会创建一个新的作用域。
      + 当调用函数时，函数体内的变量绑定位于新的函数作用域内，这些变量在函数调用结束后会销毁，并且不会影响到函数外部的变量。

3. Directory Scope。
      + 每个源代码树中的目录都有其自己的变量绑定。
      + 处理某个目录的 CMakeLists.txt 文件之前，CMake 会复制当前定义在父目录中的变量绑定来初始化新目录的作用域。
      + 在顶层目录中定义的变量对所有子目录可见，但在子目录中定义的变量对父目录不可见。

4. Persistent Cache。
      + CMake通过“缓存变量”或“缓存项”来存储值，这些值在多个项目构建周期中会持久化。
      + 缓存项的作用域是独立的，只有通过显式请求（例如使用 set() 和 unset() 命令的 CACHE 选项）才能修改。

在 CMake 中评估变量引用时，系统遵循特定的优先顺序来确定变量的值。这种顺序确保优先使用本地作用域的值，然后才考虑更全局的作用域或持久缓存的值。

1. **函数调用栈**:
      - CMake首先在当前函数调用栈中查找变量。这意味着它会查找当前函数作用域内的变量绑定，遍历嵌套函数调用（如果有的话）。

2. **当前目录作用域**:
      - 如果在函数调用栈中没有找到变量绑定，CMake接着会在当前目录作用域中查找变量绑定。这包括在任何函数调用之外设置的目录作用域中的变量。

3. **缓存条目**:
      - 如果在函数作用域和目录作用域中都没有找到变量，CMake会查找该变量的缓存条目。缓存条目在项目构建树中是持久的多次运行。如果找到了变量的缓存条目，则使用其值。

4. **未定义变量**:
      - 如果没有找到缓存条目，变量引用将评估为一个空字符串。

### 六、环境变量

环境变量与普通变量类似，但有以下区别：

1. 作用域
      - 环境变量在一个 CMake 进程中具有全局作用域。它们从不被缓存。
      
2. 引用
      - 变量引用采用 $ENV{<variable>} 的形式，使用 ENV 操作符。

3. 初始化
      - CMake 环境变量的初始值是调用进程的初始值。可以使用 `set()` 和 `unset()` 命令更改其值。这些命令仅影响正在运行的 CMake 进程，而不会影响整个系统环境。更改后的值不会写回到调用进程中，并且后续的构建或测试进程不会看到这些更改后的值。
      - 要在修改后的环境中运行命令，请参见 `cmake -E env` 命令行工具。

4. 检查
      - 要显示当前所有环境变量，请参见 `cmake -E environment` 命令行工具。  

## 七、列表

尽管 CMake 中的所有值都存储为字符串，但在某些情况下（例如在计算未引用的参数时），字符串可能会被视为列表。在这种情况下，会将字符串按 `;` 字符进行分割，但前提是这些 `;` 字符后面没有跟随一个不成对的 `[` 和 `]` 字符，并且没有立即之前带有 `\`。序列 `\;` 不会将值分割，而是在结果元素中被替换为 `;`
