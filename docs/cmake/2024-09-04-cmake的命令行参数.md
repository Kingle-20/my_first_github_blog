## 一、前言

在cmake中，很多时候是通过命令行的方式向cmake中传递参数，或者是指导cmake如何进行构建，因此有必要学习一些常用的参数，主要分为配置阶段、构建阶段、安装阶段，或者是其他有用的命令  


## 二、配置阶段  

主要用于生成构建系统的文件。

### 2.1 `-S <path>`: 指定源代码目录。  

该选项主要指定顶层的CMakeLists.txt文件所在的目录。 

+ *分离源代码和构建目录*：通过明确指定源代码目录和构建文件目录，可以确保源代码的清洁性，避免生成的构建文件混杂在源代码中。  
+ *多配置构建*：可以从同一源代码目录生成多个不同配置（如 Debug 和 Release）的构建目录。   
+ *远程构建*：在远程构建机器上，可以明确指定源代码的位置，确保构建流程的一致性和重复性。  

### 2.2 `-B <path>`: 指定生成构建文件的目录。

例如，经常将构建文件放置到build文件夹下，然后在这个目录下进行构建。  
可以选择将debug模式和realease版本放到不同目录下  

### 2.3 `-G <gennerator>`: 指定使用哪种生成器。  

主要格式为  
`cmake -G "Unix Makefiles"`  

主要参数有：

+ "Unix Makefiles"  
+ Ninja  
+ "Visual Studio 16 2019"或者"Visual Studio 15 2017"  
+ "MinGW Makefiles"  

## 三、构建阶段  

### 3.1 --build 构建阶段

语法格式：

```cmake 
cmake --build <path-to-build> [options]
```

语法格式如上，先指定了配置文件所在目录的位置，然后根据选项设置相应的功能。

+ `--target <target>`：指定要构建的目标。
+ `--config <config>`：指定构建配置（如 Debug, Release 等）。
+ `--parallel [<jobs>]`：指定并行编译的工作线程数  

## 四、安装阶段  

### 3.1 --install 

语法格式：

```cmake
cmake --install <path-to-build> [options]
```

也就是说需要使用--install选项表明这是在安装阶段的命令，它有如下常用命令选项：

+ `--prefix <path>`：指定安装路径。
+ `--config <config>`：指定要安装的构建配置。  

## 五、辅助命令  

### 5.1 -E 查看cmake支持哪些内置指令  

主要目的：获取cmake支持哪些跨平台的操作，例如打印字符串啊，或者查看环境变量等，又有不同平台有不同的命令方式，所以使用cmake就可以达到跨平台的效果，格式如下：

`cmake -E SpecificCommand CommandARG`  

相比较而言，它并不用于直接输入到命令行中，而是写道CMakeLists.txt文件中，它常常和ADD_CUSTOM_COMMAND进行联合调用。如下格式：

```cmake
ADD_CUSTOM_COMMAND(
    TARGET my-target
    COMMAND ${CMAKE_COMMAND} -E SpecificCommand CommandARG
)
```

这样可以达到跨平台效果，其中`${CMAKE_COMMAND}`表示当前执行cmake命令，它包含的完整路径的可执行文件，例如：

```cmake
message("CMAKE_COMMAND = ${CMAKE_COMMAND}")
```

就可以打印出当前cmake的位置:

```cmake
CMAKE_COMMAND = D:/Program Files/bin/cmake.exe
```

### 5.2 --version 查看版本  

## 六、特殊选项  

### 6.1 cmake -D 传递CMAKE变量值  

-D 选项用于定义 CMake 变量。你可以通过这种方式在命令行上传递变量和配置选项，而不是在 CMakeLists.txt 文件中直接设置  

主要格式为：
```cmake
cmake -DVAR_NAME=VALUE ..
```
其中，

+ -D: 标志，表示定义一个变量。
+ VAR_NAME: 变量的名称。
+ VALUE: 变量的值  

主要功能： 

+ 可以主动设置在cmakelists中使用option函数所定义的变量的值。
+ 主动设置cmake内置变量的值。

*常用的CMAKE变量值*：

+ CMAKE_INSTALL_PREFIX 设置项目的安装位置  
+ CMAKE_BUILD_TYPE 设置构建类型，常见的值包括 Debug, Release, RelWithDebInfo, 和 MinSizeRel  
+ CMAKE_C_COMPILER 和 CMAKE_CXX_COMPILER: 指定 C 和 C++ 编译器  
+ CMAKE_C_FLAGS 和 CMAKE_CXX_FLAGS: 设置传递给 C 和 C++ 编译器的标志  
+ BUILD_SHARED_LIBS: 控制是生成共享库（ON）还是静态库（off） 
+ CMAKE_MODULE_PATH: 用于指定 CMake 在执行 find_package()、find_library()、find_file()、find_path() 和 find_program() 等命令时，搜索自定义模块的额外目录  
+ CMAKE_EXPORT_COMPILE_COMMANDS: 生成用于编译命令的 JSON 文件  
+ CMAKE_PREFIX_PATH：用于指定前缀路径，CMake 会在这些前缀路径下查找已安装的软件包（包括 include、lib、和 share/cmake）  

### 6.2 --warn-uninitialized

警告未初始化的变量  

### 6.3 -P

cmake -P 选项用于运行一个指定的 CMake 脚本文件。这个选项启动 CMake 的脚本模式，这意味着 CMake 将执行脚本文件中定义的命令，而不是生成构建文件。当你使用 cmake -P <script-file> 时，CMake 会将 <script-file> 当作一个包含 CMake 命令的脚本来执行。这个脚本文件中可以包括任意合法的 CMake 命令，例如变量设置、文件操作、条件语句等。

__典型使用场景__

+ 生成配置文件: 执行脚本生成一些配置文件。
+ 文件操作: 执行脚本进行复杂的文件操作，例如复制、删除等。
+ 定制构建过程: 在构建过程中通过脚本实现一些自定义步骤。
+ 测试和实用程序: 编写简单的测试脚本或实用工具脚本，减少对外部脚本语言（如 Python 或 Bash）的依赖。  

### 6.4 -L(AH)

-L是简略输出cmakelists中的option选项，通过这个，可以知道第三方库的开发者提供给外部的有哪些选项。
-LAH就可以详细打印出其选项内容及其含义。

典型使用方式是 cmake -LAH . -B build，这将输出option对应的输出内容。

那么如何给执行的脚本传递参数呢，可以类比-DXXX="XXX-ARG"的方式传入参数，其中XXX就是参数名，而XXX-ARG就是具体的参数
