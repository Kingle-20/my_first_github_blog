## 一、前言

微软出了wsl2之后，如果不那么依赖于linux的图形界面的话，可以使用它作为虚拟机的替代产品。它的优势是你可以同时使用linux环境快速搭建的编程环境，又可以享受windows带来的丰富的其他资源。它的具体原理不是太明白，个人理解它就是运行在本地的服务器，通过一定的技术将本地磁盘挂载到linux的/mnt/目录下。

这种共享文件系统的好处是，利用windows丰富的软件资源将一些信息存储到文件中，然后再使用linux环境下自行编写程序，然后再进行处理。在ubuntu24.04中，如果需要使用matplotlib中这类可视化资源时，wsl2中的linux系统是可以在windows下弹出窗口。

一般使用范例是，使用vscode连接本地的Linux系统，其自带的终端中使用命令，便可以达到在windows环境下使用linux系统的目的。

## 二、wsl2系统安装

### 2.1 快速安装

根据[windows11 安装WSL2全流程](https://blog.csdn.net/u011119817/article/details/130745551)，对其简单总结如下：

> 第一步，安装wsl2子系统:以管理员权限打开powershell，然后依次输入以下命令:
> ``` powershell
> Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux
> # 不进行重启
> Enable-WindowsOptionalFeature -Online -FeatureName VirtualMachinePlatform
> # 进行重启
> ```

*********

> 第二步，管理员继续打开powershell，然后输入命令：
> ``` powershell
> wsl --update
> wsl --set-default-version 2
> ```

********

> 第三步，使用命令安装
> 
> ``` powershell
> # 使用这个命令看微软支持哪些操作系统
> wsl --list --online
> # 我选择的是ubuntu24.04，使用如下命令安装系统
> wsl --install -d Ubuntu-24.04
> ```

********

> 第四步，输入用户名和密码，即可创建成功了

### 2.2 修改安装位置

默认情况下使用是安装到c盘等地方，可以选择先导出到d盘，然后再重新导入即可。

1. 关闭wsl中开启的操作系统

```
wsl --shutdown
```

2. 导出发行版为 tar 文件（保存到 D 盘）

```
wsl --export <发行版名称> D:\wsl\ubuntu.tar
```

3. 注销原发行版,此操作会删除原系统（在 C 盘的那个）。

```
wsl --unregister Ubuntu-22.04
```

4. 从 tar 文件重新导入到 D 盘

```
wsl --import <发行版名称> D:\wsl\Ubuntu D:\wsl\ubuntu.tar --version 2
```

5. 设置默认用户（重要！）

使用 /etc/wsl.conf，在导入后首次启动时，先以 root 身份登录：

```
wsl -u root
```

然后编辑配置文件：

```
sudo vi /etc/wsl.conf
```

添加以下内容（假设你的用户名是 yourusername）：

```
[user]
default=yourusername
```

保存退出（esc → :wq，然后关闭 WSL：

```
exit
```

再重启 WSL：

```
wsl --shutdown
```

下次启动就会自动使用你的用户登录。

## 三、wsl2常用命令

### 3.1 查看当前环境有哪些系统

```powershell
wsl -l -v
wsl --list  --verbose
```

### 3.2 指定启动某一个系统

```
wsl -d <发行版名称>
wsl --distribution <发行版名称>
wsl # 启动默认发行版
```

### 3.4 关闭系统

法1：立即关闭某一个系统

```
wsl --terminate Ubuntu
```

法2：在已经打开的终端中输入

```
exit
```

法3：关闭所有wsl系统

```
wsl --shutdown
```

### 3.3 删除某一个系统

注意删除一个系统查看当前系统的名称是啥之后再删除，建议先关闭该系统

```powershell
wsl --unregister Debian # 卸载debian系统
wsl --unregister kali-linux # kali-linux
```

## 四、镜像源设置

由于默认安装的没有gui，因此我们需要使用命令行的方式来更改默认软件镜像源。

ubuntu24.04推荐不同镜像源使用不同的配置文件。

1. 首先在`/etc/apt/sources.list.d/`目录新建一个新的软件源，例如创建海康的是

    ```
    sudo vi /etc/apt/sources.list.d/hikvision.sources
    ```

2. 输入内容

    ```
    Types: deb
    URIs: http://mirrors.hikvision.com.cn/ubuntu/
    Suites: noble noble-security noble-updates
    Components: main restricted universe multiverse
    # Signed-By: /usr/share/keyrings/ubuntu-archive-keyring.gpg (可选)，没有可不填

    ```

3. 主要关注的内容是开源镜像网站的dists下是否有指定版本的这些目录：

    ```
    http://mirrors.hikvision.com.cn/ubuntu/dists/noble/
    http://mirrors.hikvision.com.cn/ubuntu/dists/noble-security/
    http://mirrors.hikvision.com.cn/ubuntu/dists/noble-updates/
    ```
    一般是修改一下`URIs`的值就行了。

那么，可选内容是啥意思呢？询问大模型有：

> 您在配置 Ubuntu 软件源时看到的 gpg 相关提示（如 Missing Signed-By），以及在很多开源项目、Linux 发行版、软件下载中频繁遇到的 .gpg 文件，都与 GPG（GNU Privacy Guard） 有关。
> ***********
> ✅ 一句话总结：
> GPG 是用于数字签名、加密和身份认证的安全工具。在软件分发中，它用来验证“你下载的软件确实来自官方，且未被篡改”。

> **********

> 🧩 举个生活中的例子
> 
> 想象一下：你从网上下载了一个“微信安装包”，但其实这个包是黑客伪造的，里面藏着病毒。
> 如果没有验证机制，你就无法分辨真假。
> 而 GPG 就像一个“防伪标签”：
> 官方开发者用自己的 私钥 给软件或软件列表签名。
> 你用他们的 公钥 验证签名。
> 如果验证通过 → 说明文件是官方发布的，安全可信 ✅
> 如果验证失败 → 可能被篡改，拒绝安装 ❌

一般来说，都是用公司内部镜像源了，一般也用不着验证这个。
